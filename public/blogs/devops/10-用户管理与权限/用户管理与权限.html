    # 1，用户分类

    Linux用户分为3类:
管理员: root 拥有最高权限(企业中不会直接用root登录 使用普通用户登录),root最高权限 相当于皇帝
普通用户: 以个人姓名命名的账号(提升到root权限),为了提高系统的安全性，企业都是用普通用户登录系统
虚拟用户: 启动进程必须使用一个用户称为虚拟用户，但是这个用户不需要登录操作系统。web服务 www虚拟用户,目的就是为了维持程序

系统默认UID分类:
0 # 管理员 root
1-999 # 虚拟用户(傀儡用户)
1000+ # 普通用户

可以使用命令自定义UID，可以自定义www用户uid为666的普通用户。

    # 2，passwd（用户相关配置文件）

    /etc/passwd 存放着用户信息，登录系统通过passwd文件验证登录。
root:x:0:0:root:/root:/bin/bash 以冒号分割共七列
bin:x:1:1:bin:/bin:/sbin/nologin

第一列: 用户名称  新创建的用户在文件的底部 登录系统需要验证的用户名
	如果用户前面加上#注释 则此用户无法登录
    同事离职，先别删，删了家目录下所有文件都没有了
    
第二列: 用户密码全都使用x表示,所有的用户密码在passwd中都是以x表示
	如果删除x则用户无法登录，但是可以先登录别的用户在linux系统中进行用户间切换
    正常有x切换，需要密码，删除了x就不要密码了
    
第三列: 用户的UID 身份证号  唯一标识

第四列: 用户的GID 组ID 类似小组的号码 小组的名称

第五列: 说明信息(没用)，表示用户的描述信息，是啥都行，可有可无

第六列: 家目录 管理员/root 普通用户 /home/malu

第七列: 命令解释器 /bin/bash普通用户可以登录系统，如果是/sbin/nologin虚拟用户不可以登录系统。
	Linux操作系统默认的解释器就是bash
	/bin/bash       表示可以登录操作系统，表示管理员和普通用户的，可以登录操作系统的
	/sbin/nologin   表示虚拟用户，不可以登录操作系统
	
	
示例1：统计管理员有几个，虚拟用户有几个，普通用户有几个
示例2：通过统计解释器，来统计管理员 虚拟用户 普通用户的数量
/etc/shadow        放密码的地方，root:到后面的一个冒号
/etc/group         用户组的信息
/etc/gshadow       用户组密码的信息

    # 3，创建用户

    语法格式:
	useradd [选项参数] 用户名称 # 一次只能创建一个用户
	
参数选项:
    -u # 指定uid号 身份证号 必须是唯一的
    -g # 指定gid号
    -m # 创建家目录
    -M # 不创建家目录
    -s # 指定解释器 /bin/bash/sbin/nologin
    -G #  附属组

注意:默认创建的为普通用户

案例1.创建用户，参数默认 创建了家，而且允许用户。默认创建的是普通用户
[root@malu:~]# useradd malu01
[root@malu:~]# useradd malu01 # 如果存在了会有提示
useradd: user 'malu01' already exists

[root@malu:~]# grep malu01 /etc/passwd
malu01:x:1001:1001::/home/oldboy01:/bin/bash

查看用户信息:
[root@malu:~]# id malu01
uid=1001(malu01) gid=1001(malu01) groups=1001(malu01)

详细信息: passwd
grep malu01 /etc/passwd

查看用户是否有家
[root@malu:~]# ll /home/
total 0 
oldboy drwx------ 2 malu malu 97 Jul 16 16:22
malu01 drwx------ 2 malu01 malu01 76 Jul 16 16:26


案例2.创建一个uid是666 gid666 普通账号 有家 /bin/bash www
#如果指定gid，小组必须提前创建
#如果不指定gid,小组会自动创建
第一步:指定gid先创建组
[root@malu:~]# groupadd -g666 www


案例3.创建一个uid777 gid777 不允许登录 不创建家目录的
虚拟用户 test01
第一步: 创建组
[root@malu:~]# groupadd -g777 test01
第二步: 创建用户
[root@malu:~]# useradd -u777 -g777 -M -s /sbin/nologin test01


删除用户:
userdel -r 用户名

    # 5，创建用户流程

    第一步:
useradd malu

第二步:
系统将/etc/skel/隐藏文件复制到/home/malu家目录中(复制过去的文件环境变量)

第三步:
将复制过去的文件属主属组修改为malu用户

    # 7，修改属主属级

    语法:
	chown malu.malu file # 修改文件的属主属组为malu
    chown malu file # 只修改文件的属主 了解
    chown .malu file # 只修改文件的属组 了解

参数选项:
    -R 递归修改文件的属主属组
    
案例1.授权1.txt属主属组为malu
[root@malu:~]# chown malu.malu 1.txt
[root@malu:~]# ll 1.txt
-rw-r--r-- 1 malu malu 53 Jul 12 11:42 1.txt

案例2.只改主人
[root@malu:~]# chown root 1.txt
[root@malu:~]# ll 1.txt
-rw-r--r-- 1 root oldboy 53 Jul 12 11:42 1.txt

案例3.只改组
[root@malu:~]# chown .root 1.txt
[root@malu:~]# ll 1.txt
-rw-r--r-- 1 root root 53 Jul 12 11:42 1.txt

案例4.递归修改文件属主属组
[root@malu:~]# chown -R malu.malu code/
[root@malu:~]# ll code/
total 4
drwxr-xr-x 2 malu malu 29 Jul 12 11:26 test
-rw-r--r-- 1 malu malu 18 Jul 12 11:26 www.malu.conf
[root@malu:~]# ll code/test/
total 4
-rw-r--r-- 1 malu malu 12 Jul 12 11:26 www.malu.conf

    # 11，sudo提权

    作用: 授权普通用户使用某个命令的时候，拥有root的执行权限。相当于皇帝下发尚方大宝剑。黄马褂免死 免死金牌。

语法格式:
    sudo 命令
    sudo less /var/log/messages
    sudo su - # 对su命令进行提权

第一步: 查看拥有的sudo权限(普通用户) malu
[malu@malu:~]$ sudo -l
We trust you have received the usual lecture from the local System
Administrator. It usually boils down to these three things:

    #1) Respect the privacy of others.
    #2) Think before you type.
    #3) With great power comes great responsibility.

    # 23，小结

    重点:
1.当前登录用户和文件的关系，通过关系找到对应的权限。
[malu@malu:~]$ ll /etc/hosts
-rw-r--r-- 1 root root 158 Jun 23  2020 /etc/hosts

2.rwx对应数字 421

3.修改文件权限
  ugo方式
  	     chmod ugo+x file
  	     chmod ugo-x file
  	     chmod u=x file
  	     chmod -x file
  	     chmod a+x file
  数字授权
  	     chmod 644 file
  	     chmod 600 file
  	     chmod 755 file
  递归授权: -R
    [malu@malu:~]# chmod -R 600 /code/
    [malu@malu:~]# ll -d /code/
    drw------- 3 root root 79 Jul 17 11:59 /code/
    [malu@malu:~]# ll /code/
    total 16
    -rw------- 1 root root 15 Jul 17 11:51 1.txt
    -rw------- 1 root root  2 Jul 17 11:38 2.txt
    -rw------- 1 root root 15 Jul 17 11:59 index.html
    -rw------- 1 root root 15 Jul 17 11:47 index.txt
    drw------- 2 root root 32 Jul 17 11:51 test
注意:禁止 chmod -R 777 /code  # 业务代码

4.rwx对于文件的含义
  r 可读
  w 不能干
  x 不能干
  
  rw表示可读写
  rx表示可执行
  rwx 表示可读写可执行 shell脚本才用，文件中必须是命令。
5.rwx对于目录的含义
  rwx可以进入到目录 创建 删除 改名等操作。
  rx 表示对目录下所有的文件可读
