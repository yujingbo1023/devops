    # 1，echo命令和输出重定向

    作用：echo输出内容到屏幕和文件中。

    语法结构:
    echo 字符串 回车    # 输出到屏幕
    echo 字符串 > file # 输出如到文件
    其他命令输出 >file  # 输出到文件中
    
案例1.输出malu字符串到malu.txt文件中，如果malu.txt 不存在则自动创建
[root@malu:~]# echo malu > malu.txt
[root@malu:~]# ll
total 4
-rw-r--r-- 1 root root 7 Jul  9 10:20 malu.txt
[root@malu:~]# cat malu.txt 
malu

案例2.默认> 先清空文件的内容在写入新的内容
[root@malu:~]# echo test > malu.txt
[root@malu:~]# cat malu.txt 
test

案例3.追加malu字符串到文件中
[root@malu:~]# echo malu >> malu.txt 
[root@malu:~]# cat malu.txt 
test
malu

    # 2，less命令和more命令

    作用：一页一页查看文件 查看大文件

    参数选项:
    -N 显示行号
    f 空格 往下翻页
    b 往上翻页
    1G 快速到首行
    g 快速到首行
    / 搜索内容
    n 查找下一个
    N 查找上一个
    q 退出

    less #查看文件，特别是大文件  也可以查看目录，但是不要编辑
cat vim会把整个文件，加载到内存中，文件大加载的慢
less是一次是加载一页。一页一页查看
less -N  显示行号
空格、f   一次往下翻一页 
b        往上翻页
100g     快速移动到100行
g        快速到首行
G        快速到尾巴
/        搜索内容 /number      和vim相同
n        查找下一个number      和vim相同
N        查找上一个number      和vim相同
v        进入编辑模式
q        退出less命令 如果编辑了，需要先:wq 或者:q! 回车，将文件保存或强制退出，再按q退出。

    sort命令：

    sort默认按照首字母或首数字排序，想让单词和单词挨着，把相同单词放在一起可以用
命令 参数 文件 
或者管道符接 命令 参数都行
sort -n     按照数字从小到大排序，不加是按照数字首字母
sort -n -k2 按照第2列进行排序 以空格进行分割，空格前面第一列，空格后面第二列
sort -nk2   同上 kn2不行 k2n行
sort -rn    逆序排序
sort -rnk2  在第二个根据数字从大到小排序
 
以空格或tab分割都行

    alias命令：

    系统为了安全性，给一些比较危险的命令，添加了一些参数

[root@VMware-koten ~]#alias

alias egrep='egrep --color=auto'
alias fgrep='fgrep --color=auto'

alias ll='ls -l --color=auto'
alias ls='ls --color=auto'

alias which='alias | /usr/bin/which --tty-only --read-alias --show-dot --show-tilde'

    # 5，统计文件中出现的单词数和字母数

    我有一个文件，我想知道里面出现了多少个单词，多少个字母。以/etc/passwd为例。先说如何查找单词，我们首先进行cat，查看文件。

    cat /etc/passwd        #查看该文件
    1

    发现里面有冒号，斜线，横杠，0-9的数字和字母x，所以，先把这些替换为空，通过管道符 (opens new window)接cat的数据并传递给tr（横杠是后来才发现的，所以只在代码上有体现，图片中没有显示）

    cat /etc/passwd | tr ":/[0-9]x-" " "
    1

    用xagrs -n1，把他们放到1列。

    cat /etc/passwd | tr ":/[0-9]x-" " " | xargs -n1
    1

    咱们先排个序，直接后面跟sort

    cat /etc/passwd | tr ":/[0-9]x-" " " | xargs -n1 | sort
    1

    排好序后，用uniq进行去重并统计

    cat /etc/passwd | tr ":/[0-9]x-" " " | xargs -n1 | sort | uniq -c
    1

    再用sort排一次序，通过数字排序，把出现次数多的拍到前面。搞定，统计单词，这就算结束了。

    cat /etc/passwd | tr ":/[0-9]x-" " " | xargs -n1 | sort | uniq -c | sort -rn
    1

    接下来介绍如何统计字母，两种方法，第一种。cat读取，然后tr进行替换，因为是统计字母，所以x就不进行替换了。

    cat /etc/passwd | tr ":/[0-9]-" " "
    1

    咱们先将他们排成1列。

    cat /etc/passwd | tr ":/[0-9]-" " " | xargs -n1
    1

    我们直接通过grep过滤，将匹配到的内容输出，可以用.匹配一个非换行符的字符，现在只剩字母了，也就是匹配现在所有的字符就是匹配字母了，其实grep . -o也有排序功能，之所以先排序的原因是因为用grep . -o会把空格也排序了，咱们不需要统计空格。

    cat /etc/passwd | tr ":/[0-9]-" " " | xargs -n1 | grep . -o
    1

    到这里先排序，方便去重，再去重统计，并按出现次数排序，到这里就搞定了。

    cat /etc/passwd | tr ":/[0-9]-" " " | xargs -n1 | grep . -o | sort | uniq -c | sort -rn
    1

    第二种方法是不用.进行匹配，而是用[a-z]，省略了tr替换等步骤。首先我们cat读取，然后用grep排序输出。

    cat /etc/passwd | grep [a-z] -o
    1

    到这里先排序，方便去重，再去重统计，并按出现次数排序，同样搞定！

    cat /etc/passwd | grep [a-z] -o | sort | uniq -c | sort -rn
    1

    总结如下：

    找字母及出现次数：
cat /etc/passwd | tr ":/[0-9]x-" " " | xargs -n1 | sort | uniq -c | sort -rn

找单词及出现次数：
方法1：
cat /etc/passwd | tr ":/[0-9]-" " " | xargs -n1 | grep . -o | sort | uniq -c | sort -rn
方法2：
cat /etc/passwd | grep [a-z] -o | sort | uniq -c | sort -rn
