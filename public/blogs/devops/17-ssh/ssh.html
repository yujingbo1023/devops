    # 1，SSH远程连接(基于用户名和密码)

    默认情况下，你安装完操作系统，这个系统上就已经有ssh服务了，这个服务不需要你安装，系统中自带这个服务器。使用xshell去连接服务器，也是基于ssh服务。如果把linux系统上的ssh服务关闭了，那么你的xshell也是连接不上去了。

    1.windows连接Linux系统
方法1
ssh 10.0.0.31    # 默认连接的SSH22端口,也可以连接阿里云服务器
方法2 指定用户
ssh root@10.0.0.31
方法3 指定端口
ssh root@10.0.0.31 22


修改linux的ssh服务远程连接端口 将注释去掉修改为2222
[root@nfs:~]#grep 2222 /etc/ssh/sshd_config 
Port 2222
重启生效
[root@nfs:~]#systemctl restart sshd
指定端口号连接:
ssh root@10.0.0.31 2222


2.Linux连接linux
web01连接backup服务器
[root@web01:~]#ssh 10.0.0.41
指定用户
[root@backup:~]#exit 退出远程连接
[root@web01:~]#ssh root@10.0.0.41
Authorized users only. All activities may be monitored and reported.
root@10.0.0.41's password: 
[root@backup:~]#

指定端口号:web01连接31服务器
[root@web01:~]#ssh -p2222 root@10.0.0.31

远程连接的地址可以用IP地址，域名，主机名称。
[root@nfs:~]#cat /etc/hosts
127.0.0.1   localhost localhost.localdomain localhost4 localhost4.localdomain4
::1         localhost localhost.localdomain localhost6 localhost6.localdomain6
10.0.0.41   backup

[root@nfs:~]#ping -c1 -W1 backup

通过主机名称连接:

    小结：

    SSH基于用户密码 类似屋门使用密码锁
1.windows连接linux  默认22端口
    ssh root@10.0.0.41 2222
    ssh root@host 2222
    ssh root@www.jiahua.com
2.Linux连接Linux系统
    ssh -p2222 root@10.0.0.31 
    ssh root@host 
    ssh root@www.jiahua.com
3.基础SSH的远程复制scp
    scp  源文件 root@10.0.0.41:/root/
    scp -r 目录 root@10.0.0.41:/root/

    # 2，SCP命令

    远程拷贝文件 每次都是全量。类似rsync

案例1.将web01的hosts传输到 41服务器的家目录
[root@nfs:~]#scp /etc/hosts 10.0.0.41:/root/

案例2.将41服务器家目录的hosts 拷贝到当前web服务器的/opt目录下
[root@nfs:~]#scp 10.0.0.41:/root/hosts /opt/

案例3.拷贝目录需要加 -r参数
[root@nfs:~]#scp -r /etc 10.0.0.41:/root/

scp-指定端口号
[root@nfs:~]#scp -P2222 /etc/hosts 172.16.1.31:/root/

注意用户权限的问题:
[root@nfs:~]#scp -P2222 /etc/hosts root@172.16.1.31:/root/

    # 3，免秘钥连接

    SSH免秘钥连接  类似人脸识别 指纹识别

远程连接流程:
1.远程连接目标服务器，目标服务器会返回所有已经存在的锁(默认没有)
  对手机进行开屏,手机会优先自动识别人脸，是否可以比对上。
  
2.如果本地的服务器的钥匙可以打开目标服务器上的锁，则自动连接 不需要任何验证（默认没有钥匙）
  如果人脸可以比对，则自动打开屏幕，不需要其他任何验证
  
3.没有钥匙或者有钥匙打不开锁，则目标服务器返回登录界面输入用户名和密码
  如果人脸匹配不上，返回密码的登录界面。


1.生成秘钥对(一把锁和一把钥匙) 
锁: 公钥
钥匙: 私钥
[root@web01:~]#ssh-keygen   #一路回车
[root@web01:~]#ll .ssh/   # 生成的秘钥对是在家目录下隐藏的目录.ssh目录
total 12
-rw------- 1 root root 2590 Aug  5 09:10 id_rsa			# 私钥 钥匙
-rw-r--r-- 1 root root  564 Aug  5 09:10 id_rsa.pub     # 公钥  锁 发送到目标服务器
-rw-r--r-- 1 root root  171 Aug  5 09:12 known_hosts	# 验证输入yes文件



2.将公钥传输到需要免交互连接的服务器
比如web01连接backup免秘钥，需要web01生成秘钥对，然后将公钥传输到backup服务器
[root@web01:~]#ssh-copy-id -i .ssh/id_rsa.pub 10.0.0.31
/usr/bin/ssh-copy-id: INFO: Source of key(s) to be installed: ".ssh/id_rsa.pub"
/usr/bin/ssh-copy-id: INFO: attempting to log in with the new key(s), to filter out any that are already installed
/usr/bin/ssh-copy-id: INFO: 1 key(s) remain to be installed -- if you are prompted now it is to install the new keys

Authorized users only. All activities may be monitored and reported.
root@10.0.0.31's password: 		# 输入31的root密码


#拷贝公钥简写
ssh-copy-id  10.0.0.31   # 默认拷贝~/.ssh/id_rsa.pub


3.直接远程连接测试 不需要任何验证(底层验证的是秘钥)
[root@web01:~]#ssh 10.0.0.31
[root@nfs:~]#exit
logout
Connection to 10.0.0.31 closed.
[root@web01:~]#

    # 4，SSH常见问题

    1.SSH免秘钥连接
1)生成秘钥对 ssh-keygen
2)将公钥传送到目标主机 ssh-copy-id  10.0.0.31
3)ssh 10.0.0.31
问题1.默认的.ssh目录不存在
解决方法
      a.手动创建
      b.ssh连接任意一台服务器自动创建.ssh
      c.生成秘钥对自动创建.ssh目录
[root@web01:~]#vim hehe/test.txt # 如果hehe目录不存在则无法自动创建
[root@web01:~]#

问题2.权限问题
第一个权限:
[root@web01:~]#ll -d /		# 根目录555
dr-xr-xr-x. 19 root root 250 Aug  5 11:44 /
第二个权限:  # root目录权限是550
[root@web01:~]#ll -d /root
dr-xr-x--- 5 root root 185 Aug  6 09:00 /root

第三个权限:  # .ssh目录权限是700
[root@web01:~]#ll -d /root/.ssh
drwx------ 2 root root 25 Aug  6 08:58 /root/.ssh

第四个权限: # 验证文件权限是600
[root@nfs:~]#ll .ssh/
total 8
-rw------- 1 root root 564 Aug  5 09:14 authorized_keys
