1.4 查看进程打开文件
lsof 显示所有被打开的文件,列出所有打开的文件
lsof -u username 列出特定用户打开的文件
lsof -p PID 列出特定进程打开的文件
lsof /path/to/file 列出特定文件被哪些进程打开
lsof +D /path/to/directory 列出特定目录下打开的文件
lsof -i 显示所有网络连接信息。
lsof -i :port 列出特定端口的网络连接
lsof -i tcp 列出指定协议的网络连接（如 TCP 或 UDP）
lsof > open_files.txt 查看所有打开的文件并输出到文件

选项说明
-u：指定用户。
-p：指定进程 ID。
-i：显示网络连接。
+D：递归显示指定目录下的文件。
-t：只显示进程 ID，通常用于与其他命令结合使用。

lsof -u $(whoami) 查看当前用户打开的文件
lsof -c nginx  查找某个进程（如 nginx）打开的所有文件
lsof -i @192.168.1.1  查找与特定 IP 地址相关的连接

ps aux | grep nginx
假设找到的 PID 是 1234，那么查看文件连接数：

lsof -c nginx | wc -l

1.8 limux 的启动流程
https://www.processon.com/view/link/617alee21e0853741504c0f4

Linux 的启动流程通常可以分为几个主要阶段。以下是 Linux 启动过程的简要概述：

1. BIOS/UEFI 启动
开机自检：确认硬件正常工作。

2. 引导加载程序（Bootloader）
加载引导程序：引导系统启动。

3. 内核加载
加载内核：加载程序将 Linux 内核加载到内存中，并将控制权转交给内核。
初始化内核：内核会初始化硬件、挂载根文件系统、设置内存管理等。

4. init 进程启动
启动第一个进程：内核完成初始化后，会启动第一个用户空间进程 init）。
系统初始化：init 进程会根据配置文件（如 /etc/inittab 或 systemd 的单位文件）启动其他系统服务和进程。

5. 运行级别和服务
设置运行级别：init 会设置运行级别，启动相应的服务和守护进程（如网络、图形界面等）。
systemd：在使用 systemd 的系统中，systemd 会并行启动服务，提供更快的启动时间和更好的依赖管理。

6. 用户登录
登录管理器：系统启动后，通常会启动一个登录管理器，提示用户输入用户名和密码。
用户会话：成功登录后，用户会进入一个用户会话，可以使用命令行界面（如 bash）或图形用户界面（如 GNOME、KDE 等）。

总结
Linux 的启动过程从 BIOS/UEFI 的硬件自检开始，通过引导加载程序加载内核，再由内核启动 init 进程，最终进入用户登录界面。这个过程涉及多个组件和层次，每个环节都扮演着至关重要的角色。

1.9生产环境有没有用过raid，你们公司用的raid 几?
RAID 0（条带化）
特点：将数据划分成多个块并分散存储在多个磁盘中。
优点：提高读写性能，因为可以并行访问多个磁盘。
缺点：没有冗余，任何一个磁盘故障都会导致数据丢失。
使用场景：适用于对性能要求高但对数据安全要求不高的场景。 数据库从库，存储从库，web服务器。

RAID 1（镜像）
特点：将相同的数据写入两个或多个磁盘中，形成数据的副本。
优点：提供冗余，任何一个磁盘故障都不会导致数据丢失，读性能较好。
缺点：存储效率较低（有效存储容量为总容量的一半），因为至少需要两个磁盘。
使用场景：适用于需要高数据安全性的场景。如系统盘，监控服务器。

RAID 5（带奇偶校验的条带化）
特点：将数据和奇偶校验信息分散存储在三个或三个以上的磁盘中。
优点：提供冗余，支持单盘故障，存储效率较高。
缺点：写入性能相对较低，因为需要计算和写入奇偶校验信息。
使用场景：适用于对性能和数据安全都有要求的场景。

RAID 10（1+0）
特点：结合 RAID 0 和 RAID 1，即在多个镜像对之间进行条带化。
优点：提供高性能和高冗余。
缺点：存储效率较低（有效存储容量为总容量的 50%）。
使用场景：适用于需要高性能和高可靠性的场景。高并发或高访问量，数据库主库存储。

软链接和硬链接的区别
1.硬链接inode号相同
2.软链接inode号不同
3.目录不能做硬硬链接，文件可以做硬链接，目录可以做软链接
4.硬链接不能跨文件系统，软链接可以跨文件系统
5.创建方式不同，ln 软 ln -s
6.目录默认硬链接数，文件默认硬链接数是1
7.删除硬链接不影响其他硬链接，删除软链接不影响源文件，删除源文件影响软链接变红。
8.链接数为0，和文件不被进程所占用，文件才真正的被删除。
9.硬链接和软链接都是普通文件，使用rm删除

在 Linux 操作系统中，进程的状态可以通过 ps 命令或 /proc 文件系统进行查看。进程的状态表示其在执行过程中的不同阶段。以下是 Linux 进程的主要状态及其说明：

1. 运行状态 (R - Running)
说明：进程正在运行或可运行，即在 CPU 上执行或者在就绪队列中等待 CPU 资源。
2. 可中断睡眠状态 (S - Sleeping)
说明：进程正在等待某个事件（如 I/O 操作）完成。在这个状态下，进程可以被中断（例如，通过信号）。
3. 不可中断睡眠状态 (D - Uninterruptible Sleep)
说明：进程正在等待某个事件，但在这个状态下进程无法被中断。通常是等待硬件 I/O 操作完成。
4. 停止状态 (T - Stopped)
说明：进程已被停止，可能是由于接收到信号（例如，SIGSTOP）或者由调试器控制。此状态下的进程不会执行。
5. 僵尸状态 (Z - Zombie)
说明：进程已经完成执行（退出），但其父进程尚未调用 wait() 来读取其退出状态。此时，进程的入口仍然存在于进程表中，以便父进程能够获取信息。
6. 结束状态 (X - Dead)
说明：进程已经被终止，但其信息仍然保留在系统中，以便进行清理。通常，进程一旦被完全清除，这个状态就不再出现。
7. 其他状态
I - Idle：通常指的是内核线程，表示线程处于空闲状态。
R+：表示当前进程在前台运行。
S+：表示当前进程在前台处于可中断睡眠状态。
查看进程状态
可以使用 ps 命令查看进程的状态。例如：
ps aux
          
输出中的 STAT 列会显示进程的状态。

总结
Linux 进程具有多种状态，不同的状态反映了进程的运行情况和资源的占用情况。理解这些状态能够帮助系统管理员和开发者更好地管理和调试进程。

1.23 CPU 负载，8核心，多少算满载，告警阀值大概多少?
nproc 取核心数。
lscpu 取核心数。
top 取核心数。top命令之后按1就可以看核心数。

CPU 负载是指 CPU 正在处理的进程数量和 CPU 的使用率。在 Linux 系统中，负载通常用负载平均值来表示，负载平均值是过去 1、5 和 15 分钟内等待 CPU 处理的进程数。

负载的计算
对于一个 8 核心的 CPU，满载的负载值可以简单理解为核心数的总和。具体来说：

满载状态：8 核心的 CPU，负载达到 8.0 时，表示 CPU 完全被占用。
超载状态：如果负载值超过 8.0，表示系统有更多的进程在等待 CPU 处理，可能会导致响应变慢。
告警阀值
告警阀值可以根据不同系统的性能需求、应用特性和业务场景进行调整。一般情况下，可以参考以下标准：

警告阀值：
当负载在 6.0 到 8.0 之间时，可能需要注意，尤其是当负载接近 8.0 时，可能需要进行监控和优化。
严重告警阀值：
当负载超过 8.0 时，通常需要立即采取措施，比如优化应用程序、增加更多的 CPU 核心或分配负载。
其他考量
CPU 使用率：除了负载平均值外，CPU 的实际使用率也是一个重要的指标。可以通过 top 或 htop 命令查看 CPU 的使用情况。一般来说，CPU 使用率高于 80% 时也需要引起关注。
应用特性：不同应用的负载特性可能不同。例如，某些 I/O 密集型应用可能在 CPU 负载较高的情况下仍然能正常运行，因此需要根据具体情况调整告警阀值。
长时间监控：在设置告警阀值时，最好根据系统的历史性能数据进行分析，确定合理的阀值。
总结
对于 8 核心的 CPU，负载值达到 8.0 时算作满载。一般来说，负载在 6.0 到 8.0 之间需要警惕，超过 8.0 时需要进行处理。建议结合 CPU 使用率、应用特性及系统性能数据综合评估告警阀值。
