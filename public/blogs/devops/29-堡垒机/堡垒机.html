    1，为什么需要堡垒机

    devops

    # 1，为什么需要堡垒机

    • 黑客有些时候可以操作你的系统，对于网安人员要发现黑客做了什么操作;
    • 有了堡垒机服务，可以发现入侵者在系统中做的破坏；
    • 系统运维人员会应用堡垒机监控内部人员操作行为；

    具体作用：

    • 可以拥有统一网站架构管理入口
    • 可以具有批量管理多台设备能力
    • 可以具有自动化运维能力（提高架构维护管理效率）
    • 可以具有架构服务审计监控能力

    堡垒机安全功能：4A能力

    • 身份鉴别 Authentication： 确认连接用户身份
    • 账号管理 Accounting： 需要管理多个用户
    • 权限管理 Authorization： 不同用户有不同权限
    • 安全审计 Auditing： 可以监控用户行为

    # 2，跳板机服务安装部署

    官方网站资料：https://docs.jumpserver.org/zh/v3/

    硬件：

    下载安装步骤： ce-v4.1.0版本（最新版）

    1.将下载的包上传到/opt目录下
[root@jumpserver:~]#cd /opt/
[root@jumpserver:opt]#ll
total 1082800
-rw-r--r-- 1 root root 1108784941 Aug 21 23:47 jumpserver-ce-v4.0.2-x86_64.tar.gz

2.解压安装
[root@jumpserver:opt]#tar xf jumpserver-ce-v4.0.2-x86_64.tar.gz
[root@jumpserver:jumpserver-ce-v4.0.2-x86_64]#./jmsctl.sh install
在安装过程中的一些选项都直接回车使用默认即可。

3.启动服务
[root@jumpserver:opt]#cd /opt/jumpserver-ce-v4.0.2-x86_64
[root@jumpserver:jumpserver-ce-v4.0.2-x86_64]#./jmsctl.sh start

3. Web access 访问web
http://10.0.0.81:80
Default username: admin  Default password: ChangeMe
