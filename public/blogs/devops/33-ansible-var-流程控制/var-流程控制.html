    # 1，变量的作用

    变量提供了便捷的方式来管理Ansible playbook的每一个项目中的动态值，比如nginx-1.6.3这个软件包的版本，在其它地方或许会反复使用，那么如果讲此值设置为变量，然后再在其他的playbook中调用，会方便许多。如此一来还方便维护，减少维护的成本。

    定义变量的方式：

    • 通过命令行进行变量定义
    • 在play文件中进行变量定义
    • 通过Inventory主机信息文件中进行变量定义

    变量的优先级：

    如果在定义变量时，变量冲突了

在上述的三个地方分别设置了：
1.命令行中：age=11
2.play文件中：age=12
3.Inventory中：age=13
那么，最终的age结果一定是11

变量的读取优先级为：
命令行 > playbook文件 > Inventory文件

    # 2，vars定义变量

    playbook变量可以通过多种方式进行定义，最简单的方式就是在playbook的开头通过vars进行定义。

    一个变量对应一个值:
[root@ansible:ansible]#cat test.yml
- hosts: web01
  vars:
    - pk1: wget
    - pk2: lrzsz
  tasks:
    - name: Install pakcages
      yum:
        name: "{{ pk1 }}"
        state: present
        
        
一个变量对应多个值:
[root@ansible:ansible]#cat test.yml 
- hosts: web01
  vars:
    all:
      - wget
      - lrzsz
  tasks:
    - name: Install pakcages
      yum:
        name: "{{ all }}"
        state: absent
        
        
注意: ansible调用变量如果是路径则不需要加双引号     ansible_hostname是ansible的内置变量   
[root@ansible:ansible]#cat test.yml 
- hosts: webs
  tasks:
    - name: create file
      file:
        path: /root/{{ ansible_hostname }}
        state: touch

    # 4，官方推荐的方式定义变量

    组定义变量: 
- 在当前目录创建  group_vars,比如为webs定义变量，在group_vars下创建webs，在webs文件中直接定义
单台定义变量: 
- 在当前目录创建 host_vars,比如为web01定义变量,在hosts_vars目录下创建web01,直接定义变量

单台定义:
[root@ansible:ansible]#mkdir host_vars
[root@ansible:ansible]#vim host_vars/web01
[root@ansible:ansible]#cat host_vars/web01
pk1: wget
pk2: lrzsz

调用变量:
[root@ansible:ansible]#cat test.yml
- hosts: webs
  tasks:
    - name: remove wget pack
      yum:
        name: 
          - "{{ pk1 }}" 
          - "{{ pk2 }}"
        state: present

组定义:
[root@ansible:ansible]#mkdir group_vars
[root@ansible:ansible]#vim group_vars/webs
[root@ansible:ansible]#cat group_vars/webs
pack1: wget
pack2: lrzsz

变量调用:
[root@ansible:ansible]#cat test.yml
- hosts: webs
  tasks:
    - name: remove wget pack
      yum:

          - "{{ pack1 }}"

    # 9，变量注册

    在执行shell命令时候，默认ansible不会返回结果

比如ls -l命令不会返回具体的结果信息
- hosts: web01
  tasks:
    - name: print ls -l
      command: ls -l
      
[root@ansible:ansible]#ansible-playbook  test.yml

PLAY [web01] *****************************************************************************************

TASK [Gathering Facts] *****************************************************************************************

ok: [web01]

TASK [print ls -l] *****************************************************************************************

changed: [web01]

---------------------------------------------------

比如nginx -t 检查nginx配置文件
- hosts: web01
  tasks:
    - name: nginx -t
      command: nginx -t
      
[root@ansible:ansible]#ansible-playbook  test.yml 

PLAY [web01] *****************************************************************************************

TASK [Gathering Facts] *****************************************************************************************

ok: [web01]

TASK [检查nginx配置文件] *****************************************************************************************

changed: [web01]

PLAY RECAP *****************************************************************************************

web01                      : ok=2    changed=1    unreachable=0    failed=0    skipped=0    rescued=0    ignored=0  

---------------------------------------------------

使用变量注册拿到返回的结果: register模块
将执行的结果先赋值给一个变量，然后使用debug模块输出变量中的内容。

[root@ansible:ansible]#cat test.yml
- hosts: web01
  tasks:
    - name:  检查nginx配置文件
      command: nginx -t
      register: nginx_re

    - name:  Print nginx_re 变量的内容
      debug: 
        msg: "{{ nginx_re }}"


获取内容的子级
[root@ansible:ansible]#cat test.yml
- hosts: web01
  tasks:
    - name:  检查nginx配置文件
      command: nginx -t
      register: nginx_re

    - name:  Print nginx_re 变量的内容
      debug: 
        msg: "{{ nginx_re.rc }}"

    # 10，流程控制

    案例1.判断如果客户端主机名称是web01则安装wget，其他跳过
#所有的客户端卸载wget
[root@ansible:ansible]#cat test.yml
- hosts: all
  tasks:
    - name: Install wget
      yum:
        name: wget
        state: absent


# 使用when判断 只有主机名称是web01的安装wget
[root@ansible:ansible]#cat test.yml 
- hosts: all
  tasks:
    - name: Install wget
      yum:
        name: wget
        state: present
      when: ansible_hostname == "web01"
[root@ansible:ansible]#ansible-playbook  test.yml

PLAY [all] ***********************************************************************************************************

TASK [Gathering Facts] ***********************************************************************************************
ok: [web01]
ok: [db01]
ok: [web02]
ok: [nfs]
ok: [backup]

TASK [Install wget] **************************************************************************************************
skipping: [nfs]
skipping: [db01]
skipping: [backup]
skipping: [web02]
changed: [web01]

------------------------------------------------

when判断写法，下面三种写法，换着来，有的有时候可能不好使
when: ansible_hostname == "web01"
when: ansible_hostname is search "web01"
when: ansible_hostname is match  "web01"

或者: 
when: (ansible_hostname == "web01") or (ansible_hostname == "db01")
并且:
when: (ansible_hostname == "web01") and (ansible_hostname == "db01")



[root@ansible:ansible]#cat test.yml
- hosts: all
  tasks:
    - name: Install wget
      yum:
        name: wget
        state: present
      when: ansible_hostname is search "web01"  
      
# web匹配的是包含web字符串的主机名称
[root@ansible:ansible]#cat test.yml
- hosts: all
  tasks:
    - name: Install wget
      yum:
        name: wget
        state: present
      when: ansible_hostname is search "web"
      
# match可以匹配到web的字符串执行
[root@ansible:ansible]#cat test.yml
- hosts: all
  tasks:
    - name: Install wget
      yum:
        name: wget
        state: present
      when: ansible_hostname is match "web"
[root@ansible:ansible]#ansible-playbook  test.yml

# 测试当前nginx -t执行成功后 nginx重启
[root@ansible:ansible]#cat test.yml
- hosts: web01
  tasks:
    - name: Install Nginx Server
      yum:
        name: nginx
        state: present
    - name: Configure Nginx Server
      copy:
        src: conf/nginx.conf
        dest: /etc/nginx/


    - name: check Nginx Conf
      command: nginx -t
      register: nginx_r
      ignore_errors: yes    # 忽略当前的错误继续往下执行。

    - name: Start Nginx Server
      systemd: 
        name: nginx
        state: started
        enabled: yes

    - name: Restart Nginx Server
      systemd:
        name: nginx
        state: restarted
      when:  nginx_r.rc is match "0"

        owner: "{{ item.owner }}"
        group: "{{ item.group }}"
        mode:  "{{ item.mode }}"
        state: touch
      loop:
        - {file: /root/1.txt,owner: root,group: root, mode: '0644'}
        - {file: /root/2.txt,owner: www,group: www, mode: '0600'}

    # 13，handlers

    handler用来执行某些条件下的任务，比如当配置文件发生变化的时候，通过notify触发handler去重启服务。

    在saltstack中也有类似的触发器，写法相对Ansible简单，只需要watch，配置文件即可。

    [root@ansible:ansible]#cat test.yml
- hosts: web01
  tasks:
    - name: Install Nginx Server
      yum:
        name: nginx
        state: present
    - name: Configure Nginx Server
      copy:
        src: conf/nginx.conf
        dest: /etc/nginx/
      notify: Restart Nginx Server


    - name: check Nginx Conf
      command: nginx -t
      register: nginx_r
      ignore_errors: yes

    - name: Start Nginx Server
      systemd: 
        name: nginx
        state: started
        enabled: yes

  handlers:
    - name: Restart Nginx Server
      systemd:
        name: nginx
        state: restarted
      when:  nginx_r.rc is match "0"

    注意：

    1. 无论多少个task通知了相同的handlers，handlers仅会在所有tasks结束后运行一次。

    2. Handlers只有在其所在的任务被执行时，才会被运行；如果一个任务中定义了notify调用Handlers，但是由于条件判断等原因，该任务未被执行，那么Handlers同样不会被执行。

    3. Handlers只会在每一个play的末尾运行一次；如果想在一个playbook中间运行Handlers，则需要使用meta模块来实现。例如： -meta: flush_handlers。

    4. 如果一个play在运行到调用Handlers的语句之前失败了，那么这个Handlers将不会被执行。我们可以使用meta模块的–force-handlers选项来强制执行Handlers，即使Handlers所在的play中途运行失败也能执行。

    5. 不能使用handlers替代tasks

    # 14，playbook任务标签

    默认情况下，Ansible在执行一个playbook时，会执行playbook中定义的所有任务，Ansible的标签(tag)功能可以给单独任务甚至整个playbook打上标签，然后利用这些标签来指定要运行playbook中的个别任务，或不执行指定的任务。

    打标签的方式

    1. 对一个task打一个标签
    2. 对一个task打多个标签
    3. 对多个task打一个标签

    打完标签如何使用

    1. -t: 执行指定的tag标签任务
    2. –skip-tags:执行–skip-tags之外的标签任务
    作用: 给1个或者多个task打标签，执行的时候按照标签执行
面试可能会问到。
[root@ansible:ansible]#cat test.yml 
- hosts: web01
  tasks:
    - name: Install Nginx Server
      yum:
        name: nginx
        state: present
    - name: Configure Nginx Server
      copy:
        src: conf/nginx.conf
        dest: /etc/nginx/
      notify: Restart Nginx Server


    - name: check Nginx Conf
      command: nginx -t
      register: nginx_r
      ignore_errors: yes

    - name: Start Nginx Server
      systemd: 
        name: nginx
        state: started
        enabled: yes
      tags: t_f				# 打tags 名称自定义

    - name: file create
      file:
        path: /root/tag.txt
        state: touch
      tags: t_f  			 # 打tags

  handlers:
    - name: Restart Nginx Server
      systemd:
        name: nginx
        state: restarted
      when:  nginx_r.rc is match "0"

    # 16，jinja2模块

    jinja2是Python的全功能模板引擎。

    Jinja2与Ansible啥关系

    • Ansible通常会使用jinja2模板来修改被管理主机的配置文件等…在saltstack中同样会使用到jinja2
    • 如果在100台主机上安装nginx，每台nginx的端口都不一样，如何解决？

    Ansible如何使用Jinja2

    • 使用Ansible的jinja2模板也就是使用template模块，该模块和copy模块一样，都是讲文件复制到远端主机上去，但是区别在于，template模块可以获取到文件中的变量，而copy则是原封不动的把文件内容复制过去。之前我们在推送rsync的backup脚本时，想把脚本中的变量名改成主机名，如果使用copy模块则推送过去的就是，不变，如果使用template，则会变成对应的主机名。

- hosts: webs

  tasks:
    - name: copy file env.txt web01
      template:
        src: env.txt
        dest: /root/
