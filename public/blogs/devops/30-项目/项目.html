    # 0，重要说明

    此项目所创建的虚拟机，一定要拍摄快照，后面再写其它项目时，直接恢复快照就行了。

    # 1，撰写项目实施规划文档

    小集群架构思路：

    1. 分析需求，写项目规划文档（项目经理、总监能力）

    2. 画架构图，将架构分解可以实现的模块（架构师能力）

    3. 分模块实施和测试（运维工程师，码农，程序员能力）

    4. 集群架构的服务拓展（考察自学能力）

    5. 一键自动化搞定（自动化运维能力）

    # 2，网站业务

    需要搭建的网站产品说明表格：

    产品分类 产品名 官方 运行环境
    博客 WordPress http://www.wordpress.org PHP
    知乎 wecente http://www.wecenter.com PHP
    Java 产品 zrlog JAVA、TOMCAT

    # 3，集群服务器作用、数量及规划

    集群服务器规划如下：

    服务器名字 台 作用
    负载均衡服务 2 负载均衡器，负责调度网站访问请求，需要 2 台同时做高可用。软件涉及 Nginx 和 Keepalved
    Web 服务 3 负责处理用户的请求，提供网页服务，同时数据需要备份到备份服务器，上传的数据需要放到存储服务器上。软件核心：Nginx、PHP、Tomcat，涉及 NFS 客户端、rsync 客户端、定时任务服务等
    共享存储 1 提供 blog、wecenter 等服务上的共享图片、视频、附件数据存储，软件核心是 NFS，涉及数据实时备份
    备份服务器 1 提供全网数据的备份服务，以及异地灾备、软件核心是 Rsync 服务
    管理服务器 2 管理所有服务器，包括批量分发文件、批量执行命令、批量配置服务、跳板机服务，远程拨号服务，yum 仓库服务等。涉及软件有 SSH、Ansible、跳板机、vpn 等，可自行扩展，这些服务工作中也可以分布到多台机器上。
    监控服务器 1 监控所有服务器，包括基础的 CPU、内存、磁盘、网络、以及每台主机的应用服务监控，如 Nginx、PHP-FPM、MySQL、Redis、Memcached、Tomcat 等。涉及软件 Zabbix-Server

    机器配置建议

    m01    2-4G
lb01   512m
lb02   512m
web01  512m
web02  512m
db01   1-2G
db02   1-2G
nfs    512m
backup 512m

    # 7， 项目实施规范要求

    1)所有服务都需要实现开机自启动。
2)所有服务器要做必要 linux 系统基本优化（防火墙、Selinux）。
3)软件的存放目录、脚本的存放都要规范。
   软件目录：/server/tools
   脚本目录：/server/scripts/
   站点目录：/code/{zh,blog,phpshe}
   java 站点目录：/code/jpress
   备份目录：/backup/
   存储目录：/data/
   主机名规范：见服务规划表格。
   IP 地址规范：见服务规划表格。
   域名规范：malu.com 需解析到 VIP 上
