Q43:镜像和容器的区别是啥?
定义：
	镜像：镜像是一个只读的模板，用于创建容器。它包含了运行某个应用所需的所有代码、库、依赖关系和配置文件。镜像是静态的，不会改变。
	容器：容器是镜像的一个具体实例，可以看作是镜像的运行时环境。容器是动态的，可以读写数据，并可以在运行时与其他容器或外部环境进行交互。

使用场景：
	镜像：用于分发和共享应用的软件包，可以在不同环境中保持一致性。
	容器：用于运行应用程序和服务，处理实时的操作和数据。

Q44:如何获取docker容器20分钟内产生的日志呢?
docker ps  # 查看当前运行的容器
docker logs my_container --since 20m # --since 20m 参数表示从当前时间向前推 20 分钟获取日志。

更精确的时间范围，可以使用 --since 和 --until 参数，格式为 "YYYY-MM-DD HH:MM:SS"。
docker logs my_container --since "2023-10-01 12:00:00" --until "2023-10-01 12:20:00"

如果你想实时查看日志并监控容器的输出，可以使用 -f 参数：
docker logs -f my_container

Q45:docker容器做端口映射的原理是如何的?容器内访问外网数据流程是如何的?

docker run -d -p 8080:80 my_container
8080 是主机端口，80 是容器内部的端口。这个命令的意思是将主机的 8080 端口映射到容器的 80 端口。
Docker 启动时会在主机上配置 iptables 规则，以便将进来的流量转发到容器内部。具体来说，当主机上的某个端口（如 8080）收到流量时，iptables 会将这个流量转发到相应的容器 IP 地址和端口（如 80）。同时，容器内部的服务也可以访问主机和外部网络。

工作流程
启动容器：使用 docker run 命令启动容器并指定端口映射。
配置 iptables：Docker 创建相应的 iptables 规则，将主机端口与容器端口进行绑定。
接收流量：当请求到达主机的指定端口时，iptables 规则会将请求转发到相应的容器。
容器处理请求：容器内部的服务处理请求并将响应返回给主机，最终返回到客户端。

----------------
容器内访问外网数据流程是如何的?

出站流量：
容器内的应用程序发起请求（如 HTTP 请求）到外部 IP 地址。
请求被发送到 Docker 网桥（docker0）。
Docker 使用 NAT 规则将容器的私有 IP 地址（如 172.17.x.x）转换为主机的公共 IP 地址，确保外部网络看到的是主机的 IP。

入站响应：
外部服务器将响应发送到主机的公共 IP 地址及 Docker 映射的端口。
Docker 的 NAT 规则会将响应流量重新映射到相应的容器 IP 地址，确保数据能够正确返回到发起请求的容器。

- docker容器访问外网的流程
    - 容器的eth0[容器内]
    - vethXXXX[宿主机，是容器eth0的对端设备]
    - docker0[容器的网关，vethXXXX的桥接设备]
    - ens33[需要开启内核转发功能，出去时需要走SNAT，回来时是DNAT技术，底层写了iptables规则]

Q46:docker容器有几种状态?

Created（已创建）：
容器已经创建，但尚未启动。此状态表明容器的文件系统和网络设置已准备好，但没有运行任何进程。

Running（运行中）：
容器正在运行，包含一个或多个正在执行的进程。在这个状态下，容器可以接受请求并处理任务。

Paused（已暂停）：
容器的进程被暂停，所有运行中的进程将被挂起。这个状态常用于临时停止容器而不释放其资源。可以通过 docker unpause 命令恢复运行。

Exited（已退出）：
容器的主进程已停止运行。此状态可以由多种原因引起，例如正常完成、出现错误或手动停止。可以通过 docker ps -a 命令查看退出的容器及其退出码。

Dead（已死亡）：
容器处于一种无法恢复的状态，通常是由于错误或系统问题导致的。此状态通常不常见，表示容器无法正常运行或已经崩溃。

Restarting（正在重启）：
容器正在重启中。这通常是由于设置了重启策略（如 --restart always）并且容器崩溃后自动尝试重启的情况。

Q47:"docker run"底层做了哪些事情?
执行 docker run 命令时，Docker 系统会完成以下主要步骤：
查找并拉取镜像（如必要）。
创建新的容器文件系统和 ID。
配置网络和分配 IP 地址。
设置控制组以限制资源。
挂载卷并设置文件系统选项。
初始化并执行容器主进程。
配置日志记录机制。
启动容器并监控其状态。
这些步骤确保了容器的创建、配置和运行高效且隔离，同时也提供了对资源的管理和监控。

Q48:简单陈述下虚拟机和容器的区别?最少说出5点?
架构：
虚拟机是在物理硬件上运行完整的操作系统（OS）。容器共享宿主机的操作系统内核，并在其上运行应用程序。容器是轻量级的，不需要完整的操作系统，只需包含应用程序及其依赖项。

启动时间：
虚拟机：启动整个操作系统和虚拟硬件，启动时间通常较长，可能需要几分钟。
容器启动速度非常快，通常只需几秒钟，因为它们只需启动应用程序进程。

资源利用：
虚拟机需要分配一定的资源（CPU、内存、存储），并且由于完整的操作系统，相对资源消耗较高。
容器更轻量，多个容器可以共享相同的操作系统内核，从而更高效地利用资源，允许在同一台主机上运行更多的容器。

隔离级别：
虚拟机提供了更强的隔离性，每个虚拟机都有自己的操作系统和内核，可以完全隔离不同的工作负载。
容器之间的隔离性较弱，因为它们共享宿主机内核。虽然容器在文件系统和进程空间上是隔离的，但仍存在一定的安全风险。

管理和部署：
虚拟机的管理和部署通常使用虚拟化管理工具
容器通常使用容器编排工具（如 Kubernetes）进行管理，能够更灵活地进行自动化部署和扩展，适合微服务架构。

    # 3，docker单机网络类型

    1.docker的单机网络类型有5种
- none：
    容器不分配网卡。

- host:
    使用宿主机网络，也是无需分配网卡，直接宿主机的网络，性能最高。

- bridge:
    桥接模式，会为容器分配虚拟网卡。工作机制有点类似于VM的NAT模式，
    默认就是桥接模式。

- container:
    和一个已经存在的容器共享网络。

- custom:
    单机自定义网络，内置了DNS解析功能，可以实现基于容器名称相互访问。
    可以将容器名称解析为对应的IP地址。

    # 6，docker容器的重启策略

    1.四种重启策略概述
always:
	只要容器退出，始终重启容器。
	
no:
	只要容器退出，则使用不重启容器，默认就是这种模式。

on-failure[:max]
	当容器异常退出时，就会触发重启，可以指定重启的次数，若不指定次数，则等效于"always"。

unless-stopped:
	在重启docke服务前，若容器处于Up状态，则始终会启动容器，若容器处于Exited状态，则使用不重启容器。
